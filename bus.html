Bus Travel Experience Toolkit Portal

ROLE
Solo (design & implementation)
TOOLS
Adobe Photoshop, React.js, CSS Grid, Google GEO API, Living Lab Bus RESTful API
TIMEFRAME
9 months

Background

This project was a part of the Living Lab Bus, an open development platform for public bus transportation solutions that offers:
  • Fleet of 3 electric buses (heavily supplemented with sensors) operating in Helsinki on lines 23 and 55
  • APIs that allow to fetch sensor data from buses
  • Developer Portal and “Oma kokoelma” (“Own Collection”) for apps and API access
  • Design tools (Bus Travel Experience Toolkit) based on passenger research by LLB staff

Objective

  • Find out what kind of design materials can be considered useful for developers’ ideation process
  • Find a suitable format of bus sensor data visualizations that can support and motivate developers to ideate
  • Figure out how to combine Toolkit and visualizations and integrate them into Living Lab Bus platform

Research methodology

Phase 1 - Scenario evaluation

GOAL
Exploration of Toolkit's attribute space. Discovery of the artefacts related to format and usage of the Toolkit by developers. Sensor data visualization format exploration.

PROCEDURE
Two different scenarios of Toolkit usage. Both scenarios were introduced to 10 software developers. Wireframe were created for each of the scenario to support the stories visually.

RESULTS
  • Presence of the LLB API within proposed wireframes set unnecesary expectations from the Toolkit. When introduced to an API or technology, developers most likely shift their focus towards them instead of the Toolkit.
  • Lack of easily available research data resulted in developers simply underestimating value and contents of the Toolkit.
  • Terminology and format of the Toolkit was confusing for the developers.
  • Developers expected more traditional data visualizations such as maps, parameter values, charts and so on.
